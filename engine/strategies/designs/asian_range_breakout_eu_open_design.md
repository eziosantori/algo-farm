# Asian Range Breakout EU Open — Design Report

> Generated by AlgoFarm Strategy Design Board — 2026-03-29

## Summary

**Idea:** Asian Range Breakout on US indices. During the Asian session (01:00–08:00 CET / 00:00–07:00 UTC), US index futures trade in a narrow range. A breakout at European open (08:00–11:00 CET / 07:00–10:00 UTC) signals directional momentum. Entry on close above Asian high (long) or below Asian low (short), confirmed by ATR14 > ATR28 (expanding volatility) and ADX14 > 20 (trend day). Exit on range re-entry or volatility collapse.
**Instruments:** US500, NAS100
**Timeframes:** H1
**Target:** sharpe > 0.5
**Debate rounds:** 2

## Phase 1 — Technical Analysis

**Strategy Type:** Breakout + Session-based + Momentum filter

### Indicators

| name | type | params | purpose |
|------|------|--------|---------|
| `asian_high` | session_high | `from_time: "00:00", to_time: "07:00"` | Asian session high (01:00–08:00 CET), carries forward |
| `asian_low` | session_low | `from_time: "00:00", to_time: "07:00"` | Asian session low (01:00–08:00 CET), carries forward |

| `atr_14` | atr | `period: 14` | Short-window volatility |
| `atr_28` | atr | `period: 28` | Long-window volatility (replaces invalid ATR SMA) |
| `adx_14` | adx | `period: 14` | Trend day confirmation |

### Entry Rules (LONG)
1. `eu_session > 0.5` — only trade during EU open window
2. `close > asian_high` — price breaks above Asian range high
3. `atr_14 > atr_28` — expanding volatility confirms momentum
4. `adx_14 > 20` — trending day, not range-bound

### Exit Rules (LONG — OR)
1. `close < asian_low` — price re-enters Asian range (breakout invalidated)
2. `atr_14 < atr_28` — volatility collapses

### Entry / Exit Rules SHORT — symmetric (close < asian_low entry, close > asian_high exit)

### Open Questions from Technical Analyst

- DST shift (CEST = UTC+2) will offset sessions by 1h in summer
- Trade frequency may be low (~1/week on US500 H1) — validate on real data before optimizing

### Recommended position_management

```json
  "sl_atr_mult": 2.0,

  "trailing_sl": "atr",
  "trailing_sl_atr_mult": 2.0,
  "scale_out": { "trigger_r": 1.5, "volume_pct": 50 },
  "time_exit_bars": 5,
  "trading_hours": { "from_time": "07:00", "to_time": "10:00", "force_close": true }
}
```

### Rationale
- **ATR-based SL/TP** adapts to dynamic equity index volatility (US500/NAS100 are 1.5–3× more volatile than forex)
- **R/R = 1.75** (SL 2×ATR / TP 3.5×ATR): achieves sharpe > 0.5 if win rate ≥ 45%
- **scale_out at 1.5R / 50%**: locks half position early, lets trailing stop run the rest
- **trailing_sl: atr, mult 2.0**: moves stop up as breakout extends, exits on momentum failure
- **force_close at 10:00 UTC**: prevents holding into US RTH where breakout edge dissipates
- **max_open_trades: 1**: prevents correlation stacking between US500 and NAS100

## Engine Gaps Found During Design

- **gap**: Session indicators and `trading_hours` operate in UTC; all analyst times were specified in CET
  **workaround**: Fixed CET-to-UTC offset of -1h applied (CET winter = UTC+1)
  **priority**: HIGH
  **suggested_fix**: Add optional `timezone` param to `session_high`, `session_low`, `session_active`, `TradingHours` backed by `zoneinfo`

- **gap**: DST (CEST = UTC+2) shifts session windows by 1h for ~7 months/year
  **workaround**: No automated DST compensation; strategy will drift 1h in summer
  **priority**: MEDIUM
  **suggested_fix**: Support `timezone: "Europe/Berlin"` in session params for automatic DST handling

- **gap**: `close` as OHLCV primitive not documented as valid `indicator:` reference in rules
  **workaround**: Used as `indicator: "close"` (left side) with `compare_to:` (right side) — confirmed valid
  **priority**: MEDIUM
  **suggested_fix**: Document OHLCV primitives as always-available implicit rule operands; add validation
